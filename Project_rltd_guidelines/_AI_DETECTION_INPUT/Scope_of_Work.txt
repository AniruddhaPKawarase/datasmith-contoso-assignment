This dissertation covers design, implementation, evaluation, and benchmarking of the system described above. The target is a system that beats single-agent and pipeline-decomposed baselines on enterprise supply chain workloads. The full stack is open source. Everything runs from a Docker Compose file on one laptop. Deliverables are grouped under the following heads.

Eight Complexity Dimensions. (i) ~600-table schema (Odoo ERP), (ii) federation across PostgreSQL, DuckDB, and Redis, (iii) temporal reasoning with fiscal calendars and rolling windows, (iv) hierarchical aggregation across company-region-country-warehouse-zone, (v) cross-domain joins of 5-10 tables, (vi) interactive ambiguity resolution, (vii) row-level access control with audit logging, (viii) RL self-correction with a three-attempt budget.

Five Domain-Specialist Agents. Inventory owns stock movement, quants, warehouses, and locations. Logistics owns pickings, carriers, customs, and tracking. Finance owns account moves, currency rates, tax rules, and cost centres. Demand owns sale orders, forecasts, reorder policies, and pricelists. Compliance is cross-cutting and enforces RBAC plus audit logging.

Six-Level Query Taxonomy. Level 1 is a single-table lookup. Level 2 adds a single-domain multi-table join. Level 3 spans domains plus a temporal filter. Level 4 federates across databases with currency conversion. Level 5 is predictive (e.g. supplier disruption impact). Level 6 is multi-turn conversation with carry-over context and progressive refinement.

Open-Source Stack. LangGraph orchestrates agents. OpenRouter is the primary LLM provider, with Ollama as offline fallback. Generation models: Qwen 2.5-Coder, DeepSeek-Coder, Llama 3.x. PostgreSQL with Odoo demo data; DuckDB for analytics; Redis for cache. Backend in FastAPI. Frontend in Next.js with Shadcn/ui. MLflow and DeepEval for tracking. Docker Compose for the whole thing.

Frontend. A Next.js chat interface that shows the generated SQL with syntax highlighting, the per-agent reasoning trace, the data lineage (which databases and tables were consulted), the confidence score, and the query result as a table or chart.

SCM-SQL Benchmark. 500+ annotated question-SQL pairs across the six complexity levels. Dedicated test suites for temporal reasoning, cross-domain joins, ambiguity, and multi-turn dialogue. Built on Odoo schema. Real values come from the DataCo dataset (180K+ orders).

Out of Scope. Cloud production deployment. Paid model APIs (OpenAI, Anthropic, Google). Proprietary connectors (Salesforce, SAP HANA). Fine-tuning of 70B-class models. Everything sits within local Docker on commodity hardware and within open-weights LLMs.
